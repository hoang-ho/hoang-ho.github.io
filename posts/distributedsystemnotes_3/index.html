Primary-based protocol

Horizontal partitioning. Example: Key-value store, and key and data item are hashed using consistent hashing.

  • Assume one single primary replica for each data item
  • Primary applies writes first and then propagates updates
  • Whenever a client wants to read, a request needs to be sent the server responsible for that data item. A write only happens at the primary server

Primary-backup

  • Have backup for the data at other servers
  • When a write requests is sent to a backup server, it is forwarded to the primary server. The primary server performs.

Cons: have to do more work in a write request Pros: Gain performance in a read request

Question: Can implement sequential consistent? How and Why? Why not linearizability?

Because of write acknowledgement, then when we have only one client, we have sequential consistency.

When we have two clients, we need to a FIFO channel (TCP) to guarantee that update requests are sent in order. All clients see the write in the same order. Yes we have sequential consistency.

If we have two writes from different clients, regardless whether writes are interleaving or not, the primary server will guarantee writes are in order. It’s the read requests that violate the linearizability.
